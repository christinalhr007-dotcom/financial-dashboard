# 理财师数据看板 - 部署指南

## 🚀 部署到 Streamlit Cloud（免费）

### 第一步：注册 GitHub 账号（5分钟）

1. 打开 https://github.com
2. 点击右上角 **Sign up**
3. 填写邮箱、密码、用户名，完成注册
4. 验证邮箱

### 第二步：创建 GitHub 仓库（2分钟）

1. 登录 GitHub 后，点击右上角 **+** → **New repository**
2. 填写信息：
   - Repository name: `financial-advisor-dashboard`
   - 选择 **Public**（公开，才能免费部署）
   - 点击 **Create repository**

### 第三步：上传项目文件（3分钟）

1. 在新创建的仓库页面，点击 **uploading an existing file**
2. 将以下文件/文件夹**全部**拖拽上传：
   - `app.py`
   - `data_processor.py`
   - `alert_system.py`
   - `requirements.txt`
   - `.streamlit/` 文件夹
   - `sample_data/` 文件夹
3. 点击 **Commit changes**

### 第四步：部署到 Streamlit Cloud（3分钟）

1. 打开 https://share.streamlit.io
2. 点击 **Sign in with GitHub**，用刚才的 GitHub 账号登录
3. 授权后，点击 **New app**
4. 填写信息：
   - Repository: 选择你刚创建的 `financial-advisor-dashboard`
   - Branch: `main`
   - Main file path: `app.py`
5. 点击 **Deploy!**

### 第五步：获取网址

部署完成后（约1-2分钟），你会看到：
- 网址格式：`https://financial-advisor-dashboard-你的用户名.streamlit.app`
- 这个网址可以分享给团队成员使用

## 📝 后续使用

### 更新数据
每周把新的 Excel 文件上传到 GitHub 仓库的 `sample_data/` 文件夹，Streamlit 会自动更新。

### 或者在线上传
在数据看板的界面上，有「上传文件」功能，可以直接在网页上上传新的数据文件。

---

## ❓ 常见问题

**Q: 部署失败怎么办？**
A: 检查 requirements.txt 是否正确上传，确保所有文件都在仓库根目录。

**Q: 数据不显示？**
A: 第一次使用需要上传文件，或点击「加载示例数据」按钮。

**Q: 网页打不开？**
A: Streamlit Cloud 免费版在无人访问时会休眠，首次打开需要等待10-30秒唤醒。
